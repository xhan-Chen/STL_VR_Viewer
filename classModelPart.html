Detailed Description

Represents one node in the model tree and, optionally, one STL mesh.

A ModelPart can act as a purely organisational tree node or as a renderable STL part. Renderable parts own a VTK reader, mapper, actor, optional VR actor, and filter pipeline. Parent and child pointers form the hierarchy consumed by ModelPartList.
